### PRD 0010B: Streaming via Turbo/Polling

**Goal**: Add dynamic streaming feedback by implementing a polling endpoint and appending message bubbles for prompts, responses, tokens, and phases—providing live visibility into runs.

**Scope**: Create poll endpoint; use Turbo to append chat bubbles incrementally from JSON chunks.

**Out of Scope**: Form controls/Heartbeat (0010C); audit/artifacts (0010D); LLM probes.

**Requirements**:
1) **Polling Endpoint**: GET /sap_runs/:correlation_id/status → JSON {phase, prompt, response, tokens: {used, remaining}, error?} (from SapRun or session).
2) **Bubble Append**: Use <turbo-frame id="chat-stream"> to poll/append daisyui-chat-bubble (user/system/agent classes; humanize via RAG if error).
3) **Auto-Scroll**: JS in Stimulus: after append, scrollTop = scrollHeight.
4) **Rails Guidance**: Turbo Streams for chunks (broadcast_to correlation_id); fallback poll every 3s if Turbo fails.

**Architectural Context**: Polling default (no ActionCable); vision.md: quick feedback, resilient.

**Acceptance Criteria**:
- Start run → bubbles append incrementally (phase/prompt/response/tokens).
- Error chunk → red bubble with humanized message.
- Auto-scroll keeps latest visible.
- Poll fails → retry silently (no crash).

**Manual Test Plan**:
1) Start task → wait 8s → see 2+ bubbles (e.g., "Phase: Outlining | Prompt: ...").
2) Trigger error → see red bubble "Aborted: token limit" with tokens.
3) Scroll up mid-run → auto-snaps to bottom on new bubble.

**Deliverables**: Polling controller/endpoint; Turbo partials for bubbles; Stimulus updates for poll/append/scroll.
