### PRD 0010C: Controls in Gear Menu & Heartbeat

**Goal**: Integrate per-run overrides (model/token/iter) and Heartbeat probe into the gear menu—enabling tweaks and health checks before starts.

**Scope**: Populate gear dropdown with form inputs; add Heartbeat button/endpoint for LLM probe.

**Out of Scope**: Streaming/polling (0010B); audit/artifacts (0010D).

**Requirements**:
1) **Gear Controls**: DaisyUI dropdown with select for model (Ollama/Grok/Claude—env mapped), number inputs for token limit/max iterations (defaults 1000/5); apply on close, pass to start params.
2) **Heartbeat Button**: In gear, button POST /sap_heartbeat {model} → probe "2+2?"; return JSON {alive, response, latency, error}.
3) **Feedback**: Banner bubble on probe (green "Alive: 1.8s" or red "Down: timeout").
4) **Rails Guidance**: Stimulus for menu open/close/apply; service SapAgent.heartbeat(model) for probe (real LLM call).

**Architectural Context**: Model override via SmartProxy; vision.md: no mocks, real probes.

**Acceptance Criteria**:
- Open gear → set token=4000 → submit run uses it.
- Click Heartbeat on Grok → green/red banner with latency.
- Overrides persist per-run (session store).

**Manual Test Plan**:
1) Click gear → select Grok, set token=3000 → close → start run (confirm override in logs).
2) Gear → Heartbeat → see banner "Alive: latency X" or error.
3) Switch model → repeat probe.

**Deliverables**: Gear dropdown partial; Heartbeat endpoint/service; banner rendering.
